Card

Why use a Card?

  • You have a scenario, concept or image and an associated explanation.
  • You would like the user to pause before consuming the explanation or definition.

Why not a Card?

  • You have a long scenario, question or result.
  • You don't like cards.

Format (mustache.js)

The basic structure of the control is represented as a mustache.js template. Cards are structured as having front and back content elements that contain the same possible elements on both side.

Those elements include the required body element as well as the optional header, leftAction and rightAction that correspond to the two possible buttons on the right and left of the card faces.

Both leftAction and rightAction elements contain the required sub-elements action and label that correspond to the button's class name and visible text.


						<div id="card1" class="card" data-ali="card">
							<div class="card-inner">
								<div class="card-front">
									{{#front.header}}
										<header>{{{front.header}}}</header>
									{{/front.header}}
									<div class="card-body">{{{front.body}}}</div>
									<footer>
										<div class="left-action">
											{{#front.leftAction}}
												<button class="{{front.leftAction.action}}">{{front.leftAction.label}}</button>
											{{/front.leftAction}}
										</div>
										<div class="right-action">
											{{#front.rightAction}}
												<button class="{{front.rightAction.action}}">{{front.rightAction.label}}</button>
											{{/front.rightAction}}
										</div>
									</footer>
								</div>
								<div class="card-back">
									{{#back.header}}
										<header>{{{back.header}}}</header>
									{{/back.header}}
									<div class="card-body">{{{back.body}}}</div>
									<footer>
										<div class="left-action">
											{{#back.leftAction}}
												<button class="{{back.leftAction.action}}">{{back.leftAction.label}}</button>
											{{/back.leftAction}}
										</div>
										<div class="right-action">
											{{#back.rightAction}}
												<button class="{{back.rightAction.action}}">{{back.rightAction.label}}</button>
											{{/back.rightAction}}
										</div>
									</footer>
								</div>
							</div>
						</div>

Event Model

Cards are informational and do not fire events.

Additional Features

Special Buttons

A basic informational card has two special buttons that are specifically wired to flip the card without writing any code.

A button with the class of .show-back will flip the card from front-to-back.

A button with the class of .show-front will flip the card from back-to-front.

JavaScript Control

Cards can be controlled by JavaScript without adding the show-front/-show-back buttons.


						// Show the front of the card with the id 'my_card'...
						$('#my_card').card('showfront');

						// Show the back of the card with the id 'my_card'...
						$('#my_card').card('showback');

Keep in mind that the show-back & show-front buttons are the preferred method of flipping the card. These methods are supplied for situations where card should be flipped in response to a user's action somewhere else on the page.

If you need to use this technique, be sure to use a button to trigger the action and ensure that the button has the aria-controls attribute set to the id of the card.
