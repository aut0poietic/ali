Card Carousel

Why use a Card Carousel?

  • You have multiple concepts in a single topic that are appropriate for Cards or Question Cards.
  • You'd like to quickly evaluate a user's understanding of multiple concepts or reinforce that concept.

Why not a Card Carousel?

  • You have more than 10 concepts or questions
  • You need over-arching feedback for these concepts or questions.
  • You would like to score the questions and/or provide a final score.

See Cards or Question Cards for additional limitations.

Format (mustache.js)

The basic structure of the control is represented as a mustache.js, but is essentially a wrapper around a set of cards, with each card itself wrapped in a .card-wrapper to maintain the layout.


						<div class="card-carousel" data-ali="card-carousel">
							{{#cards}}
								<div class="card-wrapper">
									{{{card}}}
								</div>
							{{/cards}}
						</div>

Style Bindings


						.card-carousel {
							/*  Carousel container */
							/*  Height is set via JavaScript to the offsetHeight
								of the tallest card */
						}
						.card-carousel > .card-wrapper {
							/*  The offset wrapper for each card that keeps
								The card in the center of the carousel */
						}
						.card-carousel > .card-wrapper[aria-hidden="false"] {
							/*  The wrapper for the currently active card */
						}
						.card-carousel > .card-wrapper.after,
						.card-carousel > .card-wrapper.before {
							/*  All other cards are either
								.before or .after the active card */
						}
						.card-carousel > .card-wrapper.after {
							/*  After cards are hidden just off-screen to the right... */
						}
						.card-carousel > .card-wrapper.after.hinted {
							/*  ...except for the .hinted card which is
								just out of view but can be moved slightly to
								'hint' that there are more cards. */
						}
						.card-carousel > .card-wrapper.before {
							/*  Before cards are hidden just off-screen to the left... */
						}
						.card-carousel > .card-wrapper.before.hinted {
							/*  ...except for the .hinted card which is
							just out of view but can be moved slightly to
							'hint' that there are more cards. */
						}

Event Model

Event: ali:complete

The complete event is fired for any Question Card that is a part of the carousel and an event for the Card Carousel itself when the last card is viewed.


						$( document ).on( 'ali:complete', $( 'dl.accordion' ),
							function ( e, data ) {
								// e: Object, the original event object
								// data: Object interaction event object
								console.log( e, data );
								// Be sure to check e.target for the source of the event
							}
						);

Data Object

Passed with the event.


						data = {
							description       : {string}     // The description of the current interaction
							id                : {string ID)  // The id of the interactions main element
							latency           : {int}        // Milliseconds the user spent viewing the interaction
							result            : {string}     // Status of the element, either "correct" or "incorrect"
							start             : {int}        // Time the user was first shown the interaction, in milliseconds
							type              : {string}     // SCORM/xAPI interaction type
							correct_responses : {array}      // An array of the correct responses as text
							learner_response  : {array}      // An array of the responses chosen by the user as text
						};

Additional Features

Special Buttons

The Card Carousel itself has no controls, and instead adds additional capabilities to cards it contains.

A card-button with the class of .show-next will navigate to the next card in the carousel.

A card-button with the class of .show-first will navigate to the first card in the carousel and flips all cards to their front face.
