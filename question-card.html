Question Card

Why use a Question Card?

  • You have a question with 1 correct answer.
  • You'd like to quickly evaluate a user's understanding of a concept or reinforce that concept.
  • Your correct and incorrect feedback contain the same information.

Why not a Question Card?

  • You have more than 4 or 5 possible responses.
  • Your responses are longer than a few words.
  • Your feedback is longer than a sentence or two or should be different for correct or incorrect.

Format (mustache.js)

The basic structure of the control is represented as a mustache.js template and is virtually identical to that of a normal card.


						<div id="card1" class="card" data-ali="card">
							<div class="card-inner">
								<div class="card-front">
									{{#front.header}}
										<header>{{{front.header}}}</header>
									{{/front.header}}
									<div class="card-body">{{{front.body}}}</div>
									<footer>
										<div class="left-action">
											{{#front.leftAction}}
												<button class="{{front.leftAction.action}}">{{front.leftAction.label}}</button>
											{{/front.leftAction}}
										</div>
										<div class="right-action">
											{{#front.rightAction}}
												<button class="{{front.rightAction.action}}">{{front.rightAction.label}}</button>
											{{/front.rightAction}}
										</div>
									</footer>
								</div>
								<div class="card-back">
									{{#back.header}}
										<header>{{{back.header}}}</header>
									{{/back.header}}
									<div class="card-body">{{{back.body}}}</div>
									<footer>
										<div class="left-action">
											{{#back.leftAction}}
												<button class="{{back.leftAction.action}}">{{back.leftAction.label}}</button>
											{{/back.leftAction}}
										</div>
										<div class="right-action">
											{{#back.rightAction}}
												<button class="{{back.rightAction.action}}">{{back.rightAction.label}}</button>
											{{/back.rightAction}}
										</div>
									</footer>
								</div>
							</div>
						</div>

Event Model

Event: ali:complete

Dispatched when the question is evaluated.


						$( document ).on( 'ali:complete', $( 'dl.accordion' ),
							function ( e, data ) {
								// e: Object, the original event object
								// data: Object interaction event objec
								console.log( e, data );
							}
						);

Data Object

Passed with the event.


						data = {
							description       : {string}     // The description of the current interaction
							id                : {string ID)  // The id of the interactions main element
							latency           : {int}        // Milliseconds the user spent viewing the interaction
							result            : {string}     // Status of the element, either "correct" or "incorrect"
							start             : {int}        // Time the user was first shown the interaction, in milliseconds
							type              : {string}     // SCORM/xAPI interaction type
							correct_responses : {array}      // An array of the correct responses as text
							learner_response  : {array}      // An array of the responses chosen by the user as text
						};

Additional Features

Special Buttons

The Question Card can utilize the same buttons as a standard Card, and adds an additional button for evaluation.

A button with the class of .evaluate-card should be created to evaluate the card and flip the card from front-to-back.

JavaScript Control

The Question Card can utilize the same JavaScript actions as a standard Card, and adds an additional method for evaluation.


						// Show the front of the card with the id 'my_card'...
						$('#my_card').card('evaluate');

Keep in mind that the evaluate buttons is the preferred method of evaluating the card.

If you need to use this technique, be sure to use a button to trigger the action and ensure that the button has the aria-controls attribute set to the id of the card.
