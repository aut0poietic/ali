Feedback Types

Any of these patters can be added to any of controls that are submitted. If more than one pattern is applied to a single control, the feedback used will be evaluated in the order below.

Dialogs

Dialogs small windows positioned in the center of the window, with a overlay covering the screen. They completely block all window interaction until they are dismissed.

Format (mustache.js)

The basic structure of the control is represented as a mustache.js template.


						{{#correct}}
							<div class="dialog-content correct">
								<h3>{{correct.title}}</h3>
								{{{correct.body}}}
							</div>
						{{/correct}}
						{{#incorrect}}
							<div class="dialog-content incorrect">
								<h3>{{incorrect.title}}</h3>
								{{{incorrect.body}}}
							</div>
						{{/incorrect}}

Dialog Style Bindings


						.dialog {
							/* Fixed outer container for the dialog
						       as well as the backdrop */
						}
						.dialog .dialog-window {
							/* Centered dialog window */
						}
						.dialog:focus .dialog-window {
							/* The dialog window when focused */
							/* Technically the *outer* window gets focus,
							   and applies the visual focus to the inner window */
						}
						.dialog .dialog-window-inner {
							/* Interior wrapper element */
						}
						.dialog .dialog-window-inner h2,
						.dialog .dialog-window-inner h3,
						.dialog .dialog-window-inner h4,
						.dialog .dialog-window-inner h5,
						.dialog .dialog-window-inner h6 {
							/* primary heading for dialog */
						}
						.dialog .dialog-window-actions {
							/* Button row at bottom of window */
						}
						.dialog[aria-hidden="true"] {
							/* Hidden; Must use display:none */
						}

Inline Feedback

Inline Feedback are content blocks positioned beneath the control. They do not block user interaction when opened. After a short animation delay, the control is given input focus.

Format (mustache.js)

The basic structure of the inline feedback is represented as a mustache.js template.

Note that the "correct" or "incorrect" classes are optional and are used by the system to display a "correct" or "incorrect" response. If the classes aren't used, the first element with the class .feedback-content is used.


						{{#correct}}
							<div class="feedback-content correct">
								<h3>{{correct.title}}</h3>
								{{{correct.body}}}
							</div>
						{{/correct}}
						{{#incorrect}}
							<div class="feedback-content incorrect">
								<h3>{{incorrect.title}}</h3>
								{{{incorrect.body}}}
							</div>
						{{/incorrect}}

Notices

Notices are very small notifications — typically only a word or two — that by default are animated out from the submit button.

Format

Notices are displayed if they there are no other feedback types in use and if the proper attributes are created on the main container.

The components add a hidden element to the control that will cause the contents of the noitce to be read by assitive technology.


						<div data-ali-notice-correct="Correct" data-ali-notice-incorrect="Incorrect">
						...
						</div>
