Answer Set

Why use an Answer Set?

  • You have questions or statements that can be classified by one of several types or labels.

Why not an Answer Set?

  • Your questions or concepts are longer.
  • Each element has a different set of potential responses.
  • You have more than 5 terms and more than 6 responses.

Format (mustache.js)

The basic structure of the control is represented as a mustache.js template.


						<div id="{{id}}" class="answer-set" data-ali="answer-set">
							{{#questions}}
								<form data-ali-correct="1" data-ali-correct-message="{{correct}}" data-ali-incorrect-message="{{incorrect}}">
									<label for="q-1">{{stem}}</label>
									<div class="answer-set-select">
										<select id="q-1">
											<option>Select...</option>
											{{#options}}
												<option>{{name}}</option>
											{{/options}}
										</select>
									</div>
									<div class="answer-set-submit">
										<button type="submit">Submit</button>
									</div>
								</form>
							{{/questions}}
						</div>

Event Model

Event: ali:complete

Dispatched when all items have been completed.


						$( document ).on( 'ali:complete', $( 'dl.accordion' ),
							function ( e, data ) {
								// e: Object, the original event object
								// data: Object interaction event objec
								console.log( e, data );
							}
						);

Event: ali:itemComplete

Dispatched when one of the questions are submitted. The $item parameter contains a jQuery reference to the form that was submitted.


						$( document ).on( 'ali:itemComplete', $( 'dl.accordion' ),
							function ( e, data, $item ) {
								// e: Object, the original event object
								// data: Object interaction event object
								// $item : the source of the event -- in this case the input element
								console.log( e, data, $item );
							}
						);

Event: ali:itemSelected

Dispatched when a select element's value is changed. The $item parameter contains a jQuery reference to the select element.


						$( document ).on( 'ali:itemSelected', $( 'dl.accordion' ),
							function ( e, data, $item ) {
								// e: Object, the original event object
								// data: Object interaction event object
								// $item : the source of the event -- in this case the input element
								console.log( e, data, $item );
							}
						);

Data Object

Passed with the event. Note the not all items will be populated for all events.

For example: latency has no meaning for an ali:itemSelected event unless the interaction was completed previously.


						data = {
							description       : {string}     // The description of the current interaction
							id                : {string ID)  // The id of the interactions main element
							latency           : {int}        // Milliseconds the user spent viewing the interaction
							result            : {string}     // Status of the element, either "correct" or "incorrect"
							start             : {int}        // Time the user was first shown the interaction, in milliseconds
							type              : {string}     // SCORM/xAPI interaction type
							correct_responses : {array}      // An array of the responses in the correct order as text
							learner_response  : {array}      // An array of the responses in the order chosen by the user as text
						};
