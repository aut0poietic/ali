Multiple Choice

Why use a Multiple Choice?

  • You have a question with 1 or more correct answers.
  • You'd like to quickly evaluate a user's understanding of a concept or reinforce that concept.

Why not a Multiple Choice?

  • You have more than 6 possible responses (or enough that they scroll off-screen).

Format (mustache.js)

The basic structure of the control is represented as a mustache.js template. Consider responses to be a list of response elements that contain the type ( checkbox or radio ), visible text, value as a lower-case letter a-z, and true or false value for correct.

Note that this example does not contain feedback. The four different feedback and how to use each type is outlined in the section on feedback types.


						<form id="{{id}}" class="multiple-choice" data-ali="multiple-choice">
							{{{stem}}}
							{{#responses}}
								<div class="input-group">
									<input type="{{type}}" id="response_{{value}}" value="{{value}}" data-correct="{{correct}}">
									<label for="response_{{value}}">{{{text}}}</label>
								</div>
							{{/responses}}
							<div class="submit-row">
								<button class="submit">Submit</button>
							</div>
						</form>

Event Model

Event: ali:complete

Dispatched when the question is evaluated.


						$( document ).on( 'ali:complete', $( 'dl.accordion' ),
							function ( e, data ) {
								// e: Object, the original event object
								// data: Object interaction event objec
								console.log( e, data );
							}
						);

Event: ali:itemSelected

Dispatched when an input is selected. The $item parameter contains a jQuery reference to the input element.


						$( document ).on( 'ali:itemSelected', $( 'dl.accordion' ),
						function ( e, data, $item ) {
							// e: Object, the original event object
							// data: Object interaction event object
							// $item : the source of the event -- in this case the input element
							console.log( e, data, $item );
							}
						);

Data Object

Passed with the event. Note the not all items will be populated for all events.

For example: latency has no meaning for an ali:itemSelected event unless the interaction was completed previously.


						data = {
							description       : {string}     // The description of the current interaction
							id                : {string ID)  // The id of the interactions main element
							latency           : {int}        // Milliseconds the user spent viewing the interaction
							result            : {string}     // Status of the element, either "correct" or "incorrect"
							start             : {int}        // Time the user was first shown the interaction, in milliseconds
							type              : {string}     // SCORM/xAPI interaction type
							correct_responses : {array}      // An array of the correct responses as text
						    learner_response  : {array}      // An array of the responses chosen by the user as text
						};
